## 一、数据库概述

### 1.1 数据库信息

| 项目 | 说明 |
|------|------|
| 数据库类型 | MySQL 8.0 |
| 字符集 | utf8mb4 |
| 排序规则 | utf8mb4_general_ci |
| 存储引擎 | InnoDB |
| 连接池 | HikariCP |

### 1.2 设计原则

1. **服务拆分**：每个微服务独立数据库，实现数据隔离
2. **逻辑删除**：使用delete_flag字段实现软删除
3. **审计字段**：统一使用create_time、update_time字段
4. **索引优化**：针对高频查询字段建立索引
5. **字段冗余**：适当冗余减少关联查询

## 四、表关系说明

### 4.1 用户相关关系

```
sys_user.id ←→ sys_user_follow.user_id (一对多)
sys_user.id ←→ sys_user_follow.follow_user_id (一对多)
sys_user.id ←→ sys_role.id (多对多，通过sys_user_role关联)
sys_role.id ←→ sys_permission.id (多对多，通过sys_role_permission关联)
```

### 4.2 版块相关关系

```
forum_category.id ←→ forum_forum.category_id (一对多)
forum_forum.id ←→ forum_moderator.forum_id (一对多)
sys_user.id ←→ forum_moderator.user_id (一对多)
```

### 4.3 帖子相关关系

```
forum_forum.id ←→ forum_post.forum_id (一对多)
sys_user.id ←→ forum_post.user_id (一对多)
forum_post.id ←→ forum_post_tag.post_id (一对多)
forum_post.id ←→ forum_post_attachment.post_id (一对多)
forum_post.id ←→ forum_comment.post_id (一对多)
forum_post.id ←→ forum_like.target_id (当target_type=1) (一对多)
forum_post.id ←→ forum_collect.post_id (一对多)
```

### 4.4 评论相关关系

```
forum_post.id ←→ forum_comment.post_id (一对多)
forum_comment.id ←→ forum_comment.parent_id (自关联，楼中楼)
sys_user.id ←→ forum_comment.user_id (一对多)
forum_comment.id ←→ forum_like.target_id (当target_type=2) (一对多)
```

## 五、索引设计

### 5.1 高频查询字段索引

| 表名 | 索引字段 | 索引类型 | 说明 |
|------|----------|----------|------|
| sys_user | username | UNIQUE | 登录查询 |

| forum_like | (target_type, target_id, user_id) | UNIQUE | 点赞状态查询 |

| 表名 | 索引字段 | 说明 |
