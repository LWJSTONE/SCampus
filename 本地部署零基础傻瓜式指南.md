## 二、安装必要软件

### 2.1 安装 JDK 1.8

#### Windows系统

1. **下载JDK**
   - 访问Oracle官网或采用开源版本：
   - 推荐下载：[Adoptium Temurin JDK 8](https://adoptium.net/temurin/releases/?version=8)
   - 选择Windows x64版本，下载.msi安装包

2. **安装JDK**
   ```
   双击下载的.msi文件
   → 点击"Next"
   → 选择安装路径（建议默认）
   → 点击"Install"
   → 等待安装完成
   ```

3. **配置环境变量**（重要！）

   **步骤1：设置JAVA_HOME**
   ```
   右键"此电脑" → 属性 → 高级系统设置 → 环境变量
   → 在"系统变量"中点击"新建"
   → 变量名：JAVA_HOME
   → 变量值：C:\Program Files\Java\jdk1.8.0_xxx（你的JDK安装路径）
   → 点击"确定"
   ```

   **步骤2：配置Path**
   ```
   在"系统变量"中找到"Path"变量
   → 点击"编辑"
   → 点击"新建"
   → 输入：%JAVA_HOME%\bin
   → 点击"确定"
   ```

4. **验证安装**
   ```
   打开命令提示符（Win+R，输入cmd）
   输入：java -version
   
   显示以下内容表示成功：
   java version "1.8.0_xxx"
   Java(TM) SE Runtime Environment...
   ```

### 2.2 安装 Maven 3.6+

1. **下载Maven**
   - 访问：https://maven.apache.org/download.cgi
   - 下载：apache-maven-3.9.6-bin.zip

2. **解压安装**
   ```
   解压到：D:\apache-maven-3.9.6（或你喜欢的路径）
   ```

3. **配置环境变量**
   ```
   新建系统变量：
   变量名：MAVEN_HOME
   变量值：D:\apache-maven-3.9.6
   
   编辑Path变量，添加：
   %MAVEN_HOME%\bin
   ```

4. **验证安装**
   ```
   cmd输入：mvn -version
   
   显示以下内容表示成功：
   Apache Maven 3.9.6...
   ```

### 2.3 安装 MySQL 8.0

1. **下载MySQL**
   - 访问：https://dev.mysql.com/downloads/mysql/
   - 选择Windows版本，下载MSI安装包

2. **安装MySQL**
   ```
   双击.msi文件
   → 选择"Developer Default"或"Server only"
   → 点击"Execute"执行安装
   → 配置MySQL Server：
     - 设置root密码（记住这个密码！）
     - 端口保持默认3306
   → 完成安装
   ```

3. **验证安装**
   ```
   打开MySQL Command Line Client
   输入root密码
   
   显示以下内容表示成功：
   mysql>
   ```

### 2.4 安装 Redis（Windows版）

1. **下载Redis**
   - 访问：https://github.com/tporadowski/redis/releases
   - 下载：Redis-x64-5.0.14.1.msi

2. **安装Redis**
   ```
   双击.msi文件
   → 选择安装路径
   → 勾选"Add Redis to PATH"
   → 点击Install
   ```

3. **启动Redis服务**
   ```
   方法1：通过服务管理器
   Win+R → services.msc → 找到Redis → 右键启动
   
   方法2：通过命令行
   cmd输入：redis-server
   
   验证：
   新开一个cmd窗口，输入：redis-cli ping
   返回PONG表示成功
   ```

### 2.5 安装 Node.js 18+

1. **下载Node.js**
   - 访问：https://nodejs.org/
   - 下载LTS版本（18.x或20.x）

2. **安装Node.js**
   ```
   双击.msi文件
   → 勾选"Automatically install the necessary tools..."
   → 点击Install
   ```

3. **验证安装**
   ```
   cmd输入：
   node -v     # 显示v18.x.x
   npm -v      # 显示9.x.x
   ```

4. **配置npm镜像（可选，加速下载）**
   ```
   npm config set registry https://registry.npmmirror.com
   ```

### 2.6 安装 MinIO（可选，用于文件存储）

1. **下载MinIO**
   - 访问：https://min.io/download
   - 下载Windows版本

2. **启动MinIO**
   ```
   在MinIO目录下执行：
   minio.exe server D:\minio-data --console-address ":9001"
   
   访问：http://localhost:9000
   默认账号密码：minioadmin / minioadmin
   ```

## 四、初始化数据库

### 4.1 打开MySQL命令行

```
方式1：MySQL Command Line Client
开始菜单 → MySQL → MySQL 8.0 Command Line Client

方式2：命令行
cmd → mysql -u root -p → 输入密码
```

### 4.2 执行SQL脚本

**方式一：命令行执行（推荐）**

打开命令提示符（cmd），进入项目doc目录，按顺序执行以下命令：

```bash
# 进入doc目录
cd D:\projects\SCampus\doc

# 1. 创建数据库（必须先执行）
mysql -u root -p --default-character-set=utf8mb4 < databases.sql

# 2. 创建表结构（创建数据库后执行）
mysql -u root -p --default-character-set=utf8mb4 < schema.sql

# 3. 导入初始数据（可选）
mysql -u root -p --default-character-set=utf8mb4 < data.sql
```

**方式二：MySQL命令行内执行**

```sql
-- 1. 创建数据库
source D:/projects/SCampus/doc/databases.sql;

-- 2. 创建表结构
SET NAMES utf8mb4;
source D:/projects/SCampus/doc/schema.sql;

-- 3. 导入初始数据
SET NAMES utf8mb4;
source D:/projects/SCampus/doc/data.sql;
```

**或者在MySQL Workbench中：**
1. 打开MySQL Workbench
2. 连接到本地MySQL
3. File → Run SQL Script → 选择对应的.sql文件

### 4.3 验证数据库

```sql
-- 查看创建的数据库
SHOW DATABASES;

-- 应该看到以下数据库：
-- forum_auth_db      (认证服务)
-- forum_user_db      (用户服务)
-- forum_category_db  (版块服务)
-- forum_post_db      (帖子服务)
-- forum_comment_db   (评论服务)
-- forum_interaction_db (互动服务)
-- forum_report_db    (审核服务)
-- forum_stats_db     (统计服务)
-- forum_notify_db    (通知服务)
-- forum_file_db      (文件服务)

-- 查看用户表数据
USE forum_user_db;
SELECT * FROM sys_user;
```

## 五、配置环境变量

### 5.1 数据库配置

项目支持通过环境变量配置数据库连接，默认配置如下：

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| MYSQL_HOST | localhost | MySQL主机地址 |
| MYSQL_PORT | 3306 | MySQL端口 |
| MYSQL_USERNAME | root | 数据库用户名 |
| MYSQL_PASSWORD | 123456 | 数据库密码 |

**方式一：修改配置文件（简单）**

直接修改各服务的 `application.yml` 文件中的数据库配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/forum_xxx_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 你的密码
```

**方式二：设置环境变量（推荐生产环境）**

Windows系统：
```cmd
set MYSQL_HOST=localhost
set MYSQL_PORT=3306
set MYSQL_USERNAME=root
set MYSQL_PASSWORD=你的密码
```

Linux/Mac系统：
```bash
export MYSQL_HOST=localhost
export MYSQL_PORT=3306
export MYSQL_USERNAME=root
export MYSQL_PASSWORD=你的密码
```

### 5.2 Redis配置

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| REDIS_HOST | localhost | Redis主机地址 |
| REDIS_PORT | 6379 | Redis端口 |
| REDIS_PASSWORD | (空) | Redis密码 |

### 5.3 JWT配置

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| JWT_SECRET | (内置默认值) | JWT密钥，生产环境必须修改 |

**生产环境务必设置JWT密钥：**
```bash
# 建议使用至少256位的随机字符串
export JWT_SECRET=your-very-long-random-secret-key-at-least-256-bits
```

### 5.4 MinIO配置（文件服务）

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| MINIO_ENDPOINT | http://localhost:9000 | MinIO服务地址 |
| MINIO_ACCESS_KEY | minioadmin | 访问密钥 |
| MINIO_SECRET_KEY | minioadmin | 秘密密钥 |
| MINIO_BUCKET_NAME | forum | 存储桶名称 |

## 九、常见问题解答

### Q1: mvn package 报错 "Could not find artifact"

**原因**：Maven无法下载依赖

**解决方案**：
```bash
# 配置阿里云镜像
# 编辑 D:\apache-maven-3.9.6\conf\settings.xml

# 在<mirrors>标签内添加：
<mirror>
  <id>aliyun</id>
  <mirrorOf>*</mirrorOf>
  <name>Aliyun Maven</name>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

### Q2: 服务启动报错 "Communications link failure"

**原因**：无法连接MySQL

**解决方案**：
1. 检查MySQL服务是否启动
2. 检查数据库连接配置（application.yml）
3. 确认数据库用户名密码正确
4. 确认数据库已创建（执行过databases.sql和schema.sql）

### Q3: 服务启动报错 "Unable to connect to Redis"

**原因**：无法连接Redis

**解决方案**：
```bash
# 检查Redis是否启动
redis-cli ping

# 如果未启动，启动Redis
redis-server
```

### Q4: 前端npm install报错

**原因**：网络问题或Node版本问题

**解决方案**：
```bash
# 切换npm镜像
npm config set registry https://registry.npmmirror.com

# 清除缓存重试
npm cache clean --force
npm install
```

### Q5: 前端页面打开空白

**原因**：后端服务未启动或网关未配置

**解决方案**：
1. 确认所有后端服务已启动
2. 检查网关服务是否正常运行
3. 查看浏览器控制台错误信息

### Q6: 登录提示密码错误

**原因**：密码已修改或数据库未初始化

**解决方案**：
```sql
-- 重置管理员密码为 admin123
USE forum_user_db;
UPDATE sys_user SET password = '$2a$10$/3ZkFMrKrB/X6wZhUit/dOthjcucMOssbCoPt7UbkNsCow0g.nevu' WHERE username = 'admin';
```

### Q7: 数据库表不存在错误

**原因**：数据库初始化脚本未执行或执行顺序错误

**解决方案**：
确保按以下顺序执行SQL脚本：
1. 先执行 `databases.sql` 创建数据库
2. 再执行 `schema.sql` 创建表结构
3. 最后执行 `data.sql` 导入初始数据

### Q8: 文件上传失败

**原因**：MinIO未启动或配置错误

**解决方案**：
1. 确认MinIO服务已启动
2. 检查forum-file服务的MinIO配置
3. 或使用本地存储模式（配置 `file.storage.type=LOCAL`）
